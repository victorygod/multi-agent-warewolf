# 狼人自爆改造计划

## 规则
白天遗言、投票以外的任意时间，狼人可以明示自己身份并出局，当前白天将立即结束并进入黑夜。

## 当前问题
自爆检查分散在 phase.js 的 5 个阶段中，通过轮询方式检查。这限制了自爆时机，前端无法主动触发。

## 改造方案

### 核心思路
自爆改为**即时中断机制**：
1. 前端随时显示自爆按钮（符合条件时）
2. 玩家点击后，后端立即标记中断状态
3. phase 循环检测到中断，跳转到公布死讯阶段

### 改动点

#### 1. GameEngine 添加中断机制

添加一个中断标记，用于接收自爆请求：
```
interrupt = null  // 正常状态
interrupt = { type: 'explode', playerId: x }  // 自爆中断
```

添加自爆处理方法：
- 验证玩家是存活的狼人
- 验证当前阶段允许自爆（非遗言、非投票后、非夜晚）
- 标记玩家死亡
- 如果警长还没选出，广播"警徽流失"消息
- 设置中断标记
- 广播自爆消息
- 返回成功

#### 2. PhaseManager 支持中断检测

在阶段循环中，每个阶段开始前检查中断标记：
- 如果有自爆中断
- 清理中断标记
- 如果当前不是"公布死讯"阶段，执行"公布死讯"阶段
- 然后进入下一天（夜晚）

注意：如果已经在公布死讯阶段自爆，不需要重复进入该阶段。

#### 3. 移除 phase 中的轮询代码

删除以下 5 处的自爆轮询：
- 警长竞选阶段
- 警长竞选发言阶段
- 警长投票阶段
- 白天讨论阶段
- 白天投票阶段

#### 4. 前端按钮逻辑

显示条件：
- 玩家是狼人
- 玩家存活
- 当前阶段不是：遗言、投票后处理、公布死讯、任何夜晚阶段

点击后发送自爆请求到后端。

### 吞警徽逻辑

警长没选出来时（game.sheriff === null），狼人自爆会导致警徽流失。

由于 game.sheriff 本来就是 null，不需要额外操作，只需在自爆消息后加一条"警徽流失"的系统消息即可。

### 中断时机

自爆是即时打断：
- 如果当前正在等待某玩家发言，取消该等待
- 如果当前正在等待投票，取消该等待
- 直接进入公布死讯阶段

### 测试要点

可以自爆的时机（白天，非遗言、非投票）：
- 警长竞选阶段
- 警长竞选发言阶段
- 警长投票阶段
- 公布死讯阶段（如果在此阶段自爆，不重复进入该阶段，直接进入夜晚）
- 白天讨论阶段
- 白天投票阶段

不可以自爆的时机：
- 遗言阶段
- 投票后处理阶段
- 任何夜晚阶段

### 文件改动

- main.js：添加中断标记和自爆处理方法
- phase.js：添加中断检测，移除轮询代码
- server.js：添加 WebSocket 路由（如果需要）
