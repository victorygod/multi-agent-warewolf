# 在线狼人杀自动结算逻辑

角色范围：村民、狼人、预言家、女巫、猎人、守卫、丘比特、白痴、警长

---

## 一、死亡判定逻辑

### 1.1 夜间死亡判定顺序

服务器按以下顺序执行死亡判定：

```
1. 狼刀结算：确定被狼人刀杀的玩家
2. 守卫守护判定：检查被刀者是否被守护
3. 女巫解药判定：检查是否使用解药，检测同守同救
4. 女巫毒药判定：毒杀不受守护影响
5. 殉情判定：情侣一方死亡，另一方跟随死亡
6. 生成最终死亡名单
```

### 1.2 同守同救判定

**规则**：当守卫守护的目标 = 女巫使用解药的目标 = 狼人刀杀的目标时，该玩家死亡（奶穿）

**服务器逻辑**：
```
IF 守卫守护了某人 AND 狼人刀杀了此人 AND 女巫解药救了此人
THEN 该玩家死亡（奶穿）
ELSE IF 守卫守护了此人 AND 狼人刀杀了此人 AND 女巫未救
THEN 该玩家存活（被守护）
ELSE IF 守卫未守护此人 AND 狼人刀杀了此人 AND 女巫解药救了此人
THEN 该玩家存活（被救活）
ELSE IF 守卫未守护此人 AND 狼人刀杀此人 AND 女巫未救
THEN 该玩家死亡（刀杀）
```

### 1.3 毒杀判定

**规则**：女巫毒药不受守卫守护影响，被毒者直接死亡

**服务器逻辑**：
```
IF 女巫使用毒药
THEN 被毒玩家死亡（无论是否被守卫守护）
```

### 1.4 殉情判定

**规则**：情侣一方死亡，另一方跟随死亡

**服务器逻辑**：
```
IF 存在情侣关系 AND 情侣一方在死亡名单中
THEN 另一方也加入死亡名单（殉情）
```

## 二、遗言权限判定

### 2.1 遗言规则（RULES.md 为准）

| 死亡情况 | 是否有遗言 |
|---------|-----------|
| 首夜死亡（刀杀/毒杀/奶穿） | 有 |
| 第二夜及以后夜间死亡 | 无 |
| 白天放逐 | 有 |
| 殉情 | 无 |
| 猎人开枪带走 | 无 |
| 白狼王自爆带走 | 无 |

**服务器逻辑**：
```
IF 死亡发生在第一夜（首夜）
THEN 有遗言
ELSE IF 死亡原因是"放逐"
THEN 有遗言
ELSE IF 死亡原因是"殉情"
THEN 无遗言
ELSE IF 死亡原因是"枪杀"（猎人开枪）
THEN 无遗言
ELSE
THEN 无遗言（第二夜及以后夜间死亡）
```

### 2.2 遗言规则核心（来自 RULES.md）

> 遗言规则核心：**只看死亡时机是首夜还是白天，不看死因**
- 首夜死亡 → 有遗言
- 白天死亡 → 有遗言
- 第二夜及以后夜间死亡 → 无遗言

## 三、翻牌规则

### 3.1 翻牌规则总览

| 角色 | 死亡原因 | 是否翻牌 | 翻牌后效果 |
|-----|---------|---------|-----------|
| 猎人 | 被刀杀 | 翻牌 | 可以开枪 |
| 猎人 | 被放逐 | 翻牌 | 可以开枪 |
| 猎人 | 被毒杀 | 不翻牌 | 直接死亡，不能开枪 |
| 猎人 | 殉情 | 不翻牌 | 直接死亡，不能开枪 |
| 白痴 | 被放逐 | 翻牌 | 免疫放逐，存活，保留发言但失去投票权 |
| 白痴 | 被刀杀 | 不翻牌 | 直接死亡 |
| 白痴 | 被毒杀 | 不翻牌 | 直接死亡 |
| 其他 | 任意 | 不翻牌 | 死亡（只在遗言时说身份） |

### 3.2 猎人翻牌判定

**服务器逻辑**：
```
IF 猎人死亡原因 = "刀杀" OR "放逐" OR "奶穿"
THEN 猎人翻牌，向所有玩家推送猎人身份
     允许猎人选择开枪目标
ELSE IF 猎人死亡原因 = "毒杀" OR "殉情"
THEN 猎人不翻牌，直接死亡
     猎人不能开枪
```

### 3.3 白痴翻牌判定

**服务器逻辑**：
```
IF 白痴死亡原因 = "放逐"
THEN 白痴翻牌，向所有玩家推送白痴身份
     白痴免疫此次放逐，存活
     白痴保留发言权利
     白痴失去投票权利
ELSE
THEN 白痴不翻牌，直接死亡
```

## 四、猎人技能判定

### 4.1 猎人开枪条件（RULES.md 为准）

**规则**：
- 被刀杀 → 可以开枪
- 被放逐 → 可以开枪
- 被毒杀 → 不能开枪
- 殉情 → 不能开枪
- 奶穿 → 可以开枪（算刀杀）

### 4.2 猎人开枪处理（RULES.md 为准）

**规则**：
- 猎人开枪必须带走一人，不能空枪
- 被带走玩家立即死亡，**无遗言**
- 被带走玩家如果是情侣，**触发殉情**（情侣另一方跟随死亡，无遗言）

**服务器逻辑**：
```
IF 猎人选择开枪
THEN 被带走玩家死亡
     该玩家无遗言
     IF 被带走玩家是情侣
     THEN 情侣另一方殉情死亡（无遗言）
     胜负判定需要等所有死亡处理完毕
```

## 五、胜负判定

### 5.1 好人胜利

**条件**：所有狼人死亡

**服务器逻辑**：
```
IF 存活狼人数量 = 0
THEN 好人胜利
```

### 5.2 狼人胜利（屠边）

**条件**：所有神职死亡 或 所有平民死亡

**服务器逻辑**：
```
IF 存活神职数量 = 0
THEN 狼人胜利（屠神）
ELSE IF 存活平民数量 = 0
THEN 狼人胜利（屠边）
```

### 5.3 第三方胜利（人狼恋）

**条件**：丘比特连接了好人+狼人（人狼恋），且情侣存活，其他人都死亡

**服务器逻辑**：
```
IF 丘比特阵营 = 第三方（人狼恋）
THEN IF 情侣都存活 AND 其他存活玩家数量 = 2
     THEN 第三方胜利
```

## 六、守卫连守限制

**规则**：守卫不能连续两晚守护同一人

**在线实现**：
- 前端在显示守卫选择UI时，直接禁用/隐藏上一夜守护的目标
- 服务器接收到的选择必然是合法的，无需再次验证

---

## 七、女巫用药限制

### 7.1 自救限制

**规则**：女巫不能自救

**在线实现**：
- 前端在显示女巫选择UI时，直接禁用女巫自己的选项
- 服务器接收到的选择必然是合法的

### 7.2 药量限制

**规则**：女巫每晚最多使用一瓶药

**在线实现**：
- 前端限制女巫每晚只能选择一个（解药或毒药），不能同时选两个
- 服务器接收到的选择必然是合法的

---

## 八、警长相关逻辑

### 8.1 票数计算

**规则**：警长投票算1.5票

**服务器逻辑**：
```
统计票数时：
IF 投票者是警长
THEN 该票计为1.5票
ELSE
THEN 该票计为1票
```

### 8.2 平票处理

**规则**：最高票者相同则为平票

**服务器逻辑**：
```
IF 最高票获得者有两人以上
THEN 进行平票PK
     平票者再次发言
     再次投票
     IF 再次平票
     THEN 警徽流失，无警长
```

---

## 九、丘比特阵营判定

**规则**：
- 两人都是好人 → 好人链，丘比特属于好人阵营
- 两人都是狼人 → 狼人链，丘比特属于狼人阵营
- 一好人一狼人 → 人狼恋，丘比特属于第三方阵营

**服务器逻辑**：
```
IF 情侣双方都是好人阵营
THEN 丘比特阵营 = 好人
ELSE IF 情侣双方都是狼人阵营
THEN 丘比特阵营 = 狼人
ELSE
THEN 丘比特阵营 = 第三方（人狼恋）
```
