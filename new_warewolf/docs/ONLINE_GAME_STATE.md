# 在线狼人杀服务器游戏状态

角色范围：村民、狼人、预言家、女巫、猎人、守卫、丘比特、白痴、警长

---

## 服务器需要时刻记录的全部游戏状态

服务器在游戏进行过程中需要实时维护以下所有状态信息：

---

## 一、基础游戏信息

```
1. 当前是第几天（1、2、3...）
2. 当前游戏阶段（大阶段：等待/夜晚/白天/结束）
3. 当前游戏子阶段（夜晚：守卫/狼人/女巫/预言家；白天：警长竞选/讨论/投票等）
4. 游戏板子配置（多少狼人、什么神职）
5. 游戏是否已结束
6. 胜利方（好人/狼人/第三方）
```

---

## 二、所有玩家状态（每个玩家都需要记录）

```
对每个玩家（1-12号座位）：
1. 玩家ID（连接标识）
2. 玩家名称
3. 连接状态（在线/离线/断线）
4. 身份（村民/狼人/预言家/女巫/猎人/守卫/丘比特/白痴）
5. 阵营（好人/狼人/第三方）
6. 是否存活（存活/死亡）
7. 死亡原因（刀杀/毒杀/奶穿/殉情/放逐/枪杀）
8. 是否有遗言权限
9. 是否已翻牌（猎人/白痴）
10. 是否可以投票
11. 是否可以发言
12. 是否是警长
13. 是否是情侣
14. 情侣的另一方座位号（如果是情侣）
15. 是否是狼人（狼人之间互相知道身份，可夜间群聊）
16. 是否是丘比特
```

---

## 三、存活状态

```
1. 存活玩家列表（哪些座位号还活着）
2. 死亡玩家列表（哪些座位号已死亡）
3. 存活狼人数量
4. 存活神职数量（预言家、女巫、猎人、守卫、白痴）
5. 存活平民数量（村民）
```

---

## 四、夜晚行动状态（每晚更新）

**夜晚阶段执行顺序（统一）：**
```
丘比特（仅第一夜）→ 守卫 → 狼人 → 女巫 → 预言家 → 夜晚结算
```

```
1. 守卫：
   - 今晚守护了谁
   - 上一晚守护了谁（用于前端禁用连守选项）

2. 狼人：
   - 今晚刀杀了谁
   - 狼人之间是否已确认刀杀目标

3. 女巫：
   - 今晚是否使用了解药，解药救了什么人
   - 今晚是否使用了毒药，毒药毒了什么人
   - 解药是否已用完
   - 毒药是否已用完

4. 预言家：
   - 今晚查验了什么人
   - 查验结果是什么（好人/狼人）
   - 历史查验记录（验过谁，结果是什么）
```

---

## 五、投票状态

```
1. 警长竞选：
   - 第一天是否已进行警长竞选
   - 警长是谁
   - 上警的玩家列表
   - 警下投票记录

2. 放逐投票：
   - 投票是否已开始
   - 已投票的玩家列表
   - 今天的投票记录（谁投了谁）
   - 昨天及之前的投票记录
   - 警长投票权重（1.5票）
```

---

## 六、发言状态

```
1. 当前发言阶段是否在进行
2. 当前轮到谁发言（座位号）
3. 发言顺序队列（按顺序排列的座位号列表）
4. 当前是第几个发言
5. 发言是否已结束
6. 警长是否已指定发言顺序
7. 发言模式：自由发言/按顺序发言
```

---

## 七、消息历史（带可见性蒙版）

```
每条消息记录：
1. 消息ID（唯一标识）
2. 发送者座位号
3. 消息内容
4. 发送时间
5. 消息类型：
   - 白天发言（全存活玩家可见）
   - 遗言（全存活玩家可见）
   - 警长竞选发言（全存活玩家可见）
   - 狼人夜间聊天（仅狼人可见）
   - 情侣聊天（仅情侣双方可见）
   - 系统消息（全员可见，如"天亮了"）
6. 可见性范围：
   - 所有人
   - 仅狼人
   - 仅情侣
   - 仅丘比特
   - 仅预言家（查验结果）
   - 仅女巫（被刀信息）
   - 仅守卫（守护目标）
   - 仅自己（私有状态）
```

**消息可见性规则：**
- 白天发言/遗言/警长竞选发言 → 全员可见
- 狼人夜间聊天 → 仅狼人可见
- 情侣聊天 → 仅情侣双方可见
- 丘比特连接结果 → 仅丘比特可见
- 预言家查验结果 → 仅预言家可见
- 女巫被刀信息 → 仅女巫可见
- 守卫守护结果 → 仅守卫可见
- 玩家私有状态（身份、阵营）→ 仅自己可见

---

## 八、死亡记录

```
记录每一条死亡信息：
1. 死亡发生在第几天（首夜/第一夜/第二夜...）
2. 死亡玩家的座位号
3. 死亡原因（刀杀/毒杀/奶穿/殉情/放逐/枪杀）
4. 是否有遗言
5. 是否有翻牌（猎人/白痴）
6. 猎人是否开枪，开枪带走了谁
```

---

## 九、特殊身份状态

```
1. 警长：
   - 警长是谁
   - 警长是否已死亡
   - 警徽是否已流失
   - 如果警长死亡，是否已传警徽，传给谁

2. 丘比特：
   - 丘比特是否在游戏中
   - 丘比特的座位号
   - 丘比特连接的情侣座位号（两个）
   - 丘比特属于哪个阵营（好人链/狼人链/第三方）

3. 情侣：
   - 情侣关系（座位号A ↔ 座位号B）
   - 情侣是否都存活
   - 是否已触发殉情
```

## 九点五、遗言阶段特殊处理状态

```
当有玩家死亡时，遗言阶段需要同时处理以下状态：

1. 警徽流转状态：
   - 警长是否在夜间死亡
   - 警长是否选择传警徽
   - 警徽传给谁（如果选择传）
   - 是否选择不传（警徽流失）

2. 猎人技能状态：
   - 猎人是否死亡
   - 猎人死亡原因（刀杀/放逐可开枪，毒杀/殉情不能开枪）
   - 猎人是否已选择开枪
   - 猎人开枪带走了谁
   - 如果猎人开枪，被带走的人没有遗言

3. 白痴技能状态：
   - 白痴是否被放逐
   - 白痴是否已翻牌免疫
   - 白痴翻牌后：保留发言权，失去投票权

4. 殉情状态：
   - 情侣一方死亡后，另一方是否殉情
   - 殉情者是否有遗言（无）

5. 遗言权限判断结果：
   - 每个死亡玩家是否有遗言
   - 遗言是否已发表完成
```

**死亡处理顺序（RULES.md 为准）：**
```
1. 公布死亡名单
2. 有遗言权限的玩家发表遗言
3. 判断警长是否需要传警徽（如警长夜间死亡）
4. 猎人翻牌（如猎人死亡且可开枪）
5. 猎人选择开枪目标，被带走人加入死亡队列（无遗言）
6. 处理殉情（如情侣一方死亡，包括猎人开枪带走的情侣）
7. 判断胜负
```

**遗言权限规则（RULES.md 为准）：**
- **有遗言**：首夜死亡（刀杀/毒杀/奶穿）、白天放逐
- **无遗言**：第二夜及以后夜间死亡、殉情、猎人开枪带走、白狼王自爆带走

**猎人开枪规则（RULES.md 为准）：**
- 被刀杀/放逐/奶穿 → 可以开枪
- 被毒杀/殉情 → 不能开枪
- 猎人开枪必须带走一人，不能空枪
- 猎人开枪带走的玩家无遗言
- 猎人开枪带走情侣后，触发殉情（情侣另一方跟随死亡，无遗言）

**白痴翻牌规则（RULES.md 为准）：**
- 被放逐 → 翻牌免疫，存活，保留发言权，失去投票权
- 被刀杀/毒杀 → 不翻牌，直接死亡

```

- 上一夜守护的目标

女巫：
- 解药是否已用
- 毒药是否已用

猎人：
- 猎人是否已死亡
- 猎人是否可以开枪

丘比特：

- 丘比特属于哪个阵营
```

## 十一、警长竞选阶段状态（第一天）

```
警长竞选当前子阶段：
1. 未开始
2. 上警中（玩家可以点击上警）
3. 竞选发言中（按顺序发言）
4. 退水中（玩家可以点击退水）
5. 投票中（警下玩家投票）
6. 平票PK中
7. 已结束（警长产生或警徽流失）

上警玩家列表
警下玩家列表
平票玩家列表（用于PK）
```

---

## 十二、游戏流程控制状态

```
1. 当前阶段是否可以操作
2. 当前可以操作的角色列表
3. 是否在等待玩家确认
4. 阶段开始时间（用于记录游戏进度）
```

- 游戏状态：进行中

玩家状态：
- 1号：预言家，死亡，刀杀，有遗言
- 2号：村民，存活，在线
- 3号：狼人，存活，在线

- 6号：守卫，存活，在线
- 7号：狼人，死亡，刀杀
- 8号：村民，存活，在线
- 9号：白痴，存活，在线
- 10号：狼人，存活，在线
- 11号：村民，存活，离线
- 12号：狼人，死亡，刀杀

存活状态：
- 存活：2,3,4,5,6,8,9,10（8人）
- 死亡：1,7,12（3人）
- 存活狼人：3,10（2人）
- 存活神职：4,5,6,9（4人）

- 守卫：守护5号，上一晚守护6号

- 首夜：无死亡
- 第一夜：7号（狼人），刀杀
- 第二夜：1号（预言家），刀杀，有遗言

- 本局无丘比特
