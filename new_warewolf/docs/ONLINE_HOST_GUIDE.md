# 在线狼人杀服务器主持指南

角色范围：村民、狼人、预言家、女巫、猎人、守卫、丘比特、白痴、警长

---

## 一、发牌阶段

**服务器动作：**
```
1. 根据板子配置生成身份池，随机分配给各座位
2. 记录所有玩家身份（服务器底牌）
3. 向每个玩家推送私有状态：
   - 自己的身份
   - 自己的阵营（好人/狼人/第三方）
   - 特殊关系（如是情侣，告知另一方座位号）
```

---

## 二、夜晚阶段

### 2.1 丘比特行动（仅第一夜，如有丘比特）

**服务器动作：**
```
1. 向丘比特推送选择目标UI
2. 接收丘比特选择的两名玩家
3. 记录情侣关系
4. 判定情侣阵营：
   - 两人都是好人 → 好人链，丘比特属于好人阵营
   - 两人都是狼人 → 狼人链，丘比特属于狼人阵营
   - 一好人一狼人 → 人狼恋（第三方）
5. 向丘比特推送情侣关系（注意：丘比特不知道情侣的具体身份，只知道座位号）
6. 向两名情侣玩家推送：你们是情侣，告知对方座位号
   - 注意：情侣只知道对方是情侣，不知道对方身份和阵营
```

---

### 2.2 守卫守护阶段

**服务器动作：**
```
1. 向守卫推送选择守护目标UI
   - 注意：前端应禁用/隐藏上一夜守护的目标（不能连守）
2. 接收守卫选择
3. 记录守护目标
```

---

### 2.3 狼人刀杀阶段

**服务器动作：**
```
1. 向所有狼人推送：狼人行动阶段开始
2. 开启狼人聊天频道，接收狼人打字聊天，转发给所有狼人
3. 接收狼人的刀杀目标选择
4. 记录刀杀目标
5. 关闭狼人聊天频道
```

---

### 2.4 女巫用药阶段

**服务器动作：**
```
1. 向女巫推送：
   - 被刀者座位号（女巫可以看到谁死了）
   - 解药是否还有
   - 毒药是否还有
   - 选择UI（救/不救，毒/不毒）
2. 接收女巫的选择
3. 检查限制：
   - 自救限制：被刀者是女巫自己 → 拒绝，提示不能自救
   - 药量限制：解药和毒药都选了 → 拒绝，提示每晚只能用一瓶药
4. 记录用药情况
```

---

### 2.5 预言家查验阶段

**服务器动作：**
```
1. 向预言家推送选择查验目标UI
2. 接收预言家选择
3. 查询目标实际身份
4. 向预言家推送查验结果：好人或狼人
5. 记录查验结果
```

---

### 2.6 夜晚结算阶段

**服务器动作：**
```
1. 汇总所有夜间行动
2. 执行死亡判定：
   - 狼刀目标 + 守卫守护 + 女巫解药 → 同守同救判定
   - 女巫毒药 → 毒杀判定
   - 情侣一方死亡 → 殉情判定
3. 生成死亡名单
4. 判断每个死者的遗言权限：
   - 首夜死亡 → 有遗言
   - 第二夜及以后死亡 → 无遗言
5. 向所有玩家推送死亡结果（或平安夜）
6. 判断警徽流转：
   - IF 警长在夜间死亡 → 向警长推送是否传警徽UI
   - IF 警长选择传警徽 → 指定继任者
   - IF 警长选择不传 → 警徽流失
```

---

## 三、白天阶段（第一天）

### 3.1 警长竞选（仅第一天，最先进行）

**阶段1：上警**
```
服务器动作：
1. 向所有玩家推送"警长竞选开始"
2. 接收玩家点击"上警"按钮
3. 记录上警玩家列表
4. 向所有玩家推送上警玩家列表
```

**阶段2：竞选发言**
```
服务器动作：
1. 按顺序向每个上警玩家推送发言UI
2. 接收玩家发言内容
3. 广播发言给所有玩家
```

**阶段3：退水**
```
服务器动作：
1. 向所有玩家推送"退水阶段开始"
2. 接收玩家点击"退水"按钮
3. 更新上警玩家列表
4. 向所有玩家推送更新后的列表
```

**阶段4：投票**
```
服务器动作：
1. 向警下玩家（未上警者）推送投票UI
2. 接收投票
3. 统计票数（警长1.5票）
4. 判断是否平票
```

**阶段5：平票PK**
```
服务器动作：
IF 最高票者票数相同：
  1. 向平票者推送PK发言UI
  2. 接收PK发言
  3. 广播PK发言
  4. 再次投票
  5. IF 再次平票 → 警徽流失，无警长
```

**阶段6：警长确定**
```
服务器动作：
1. 推送警长结果（警长座位号）
2. 记录警长
```

---

### 3.2 天亮公布

**服务器动作：**
```
1. 向所有玩家推送"天亮了"
2. 推送昨晚死亡名单，或"昨晚是平安夜"
```

---

### 3.3 遗言处理（首夜死亡玩家）

**服务器动作：**
```
1. 对有遗言权限的死亡玩家，推送遗言UI
2. 接收遗言内容
3. 广播遗言给所有玩家
4. 死亡玩家报出身份（遗言时说出，不翻牌）
```

---

### 3.4 白天讨论

**服务器动作：**
```
1. 警长指定发言顺序
2. 按顺序向玩家推送发言UI
3. 接收玩家发言
4. 广播发言给所有玩家
```

---

### 3.5 放逐投票

**服务器动作：**
```
1. 向所有存活玩家推送投票UI
2. 接收玩家投票
3. 统计票数（警长1.5票）
4. 判断是否平票
5. 推送投票结果
```

**平票处理：**
```
IF 平票：
  方案A：平票者PK发言后再次投票
  方案B：再次平票 → 平安日（无人被放逐）
```

---

### 3.6 放逐后处理

**服务器动作：**
```
1. 向被放逐者推送遗言UI
2. 接收遗言
3. 广播遗言
4. 被放逐者报出身份（不说出具体身份，只说有遗言）
5. 判断特殊技能：
   - IF 被放逐者是猎人：
     * 猎人翻牌（向所有玩家推送猎人身份）
     * 向猎人推送开枪UI
     * 接收开枪目标
     * 被带走玩家死亡，无遗言
   - IF 被放逐者是白痴：
     * 白痴翻牌（向所有玩家推送白痴身份）
     * 白痴免疫此次放逐，存活
     * 保留发言权利
     * 失去投票权利
6. 判断警徽流转：
   - IF 警长被放逐 → 向警长推送是否传警徽UI
   - IF 警长选择传警徽 → 指定继任者
   - IF 警长选择不传 → 警徽流失
```

## 五、胜负判定

**服务器动作：**
```
每个白天/夜晚结束后：
1. 统计存活狼人数
2. 统计存活神职数（预言家、女巫、猎人、守卫、白痴）
3. 统计存活平民数
4. 检查胜利条件：
   - IF 存活狼人 = 0 → 好人胜利
   - IF 存活神职 = 0 → 狼人胜利（屠神）
   - IF 存活平民 = 0 → 狼人胜利（屠边）
   - IF 人狼恋存在 AND 情侣存活 AND 其他人都死亡 → 第三方胜利
5. 推送胜利结果，公布所有玩家身份
```

## 六、特殊场景处理

### 6.1 狼人自爆

**服务器动作：**
```
1. 接收狼人自爆信号
2. 该玩家翻牌死亡
3. 判断发生阶段：
   - IF 在警长竞选阶段 → 警徽流失
   - IF 在白天发言阶段 → 立即进入黑夜
   - IF 在投票阶段 → 立即进入黑夜
4. 立即进入黑夜
5. IF 首夜有死亡 → 仍有遗言
```

## 七、翻牌规则

**规则**：
- 猎人：被刀杀或被放逐时翻牌，可以开枪
- 白痴：被放逐时翻牌，免疫放逐
- 其他所有角色：死亡时不翻牌（只在遗言时说出身份）

**服务器逻辑**：
```
IF 死亡原因是"刀杀" AND 死亡者是猎人
THEN 猎人翻牌，可以开枪

IF 死亡原因是"放逐" AND 死亡者是猎人
THEN 猎人翻牌，可以开枪

IF 死亡原因是"放逐" AND 死亡者是白痴
THEN 白痴翻牌，免疫放逐，存活

其他所有情况：不翻牌
```
