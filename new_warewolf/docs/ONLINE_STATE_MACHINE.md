# 在线狼人杀状态机与阶段转换

角色范围：村民、狼人、预言家、女巫、猎人、守卫、丘比特、白痴、警长

---

## 一、整体状态机

```
游戏状态：
├── WAITING      （等待玩家加入）
├── PLAYING      （游戏进行中）
│   ├── NIGHT    （夜晚阶段）
│   └── DAY      （白天阶段）
└── ENDED        （游戏结束）

游戏开始后，服务器自动在 NIGHT 和 DAY 之间循环，
每个阶段内部按顺序执行各子阶段，直到分出胜负。
```

---

## 二、夜晚阶段流程（统一顺序）

```
夜晚阶段开始
    │
    ▼
┌─────────────────────────────────────┐
│ 阶段1：丘比特（仅第一夜，如有）    │
├─────────────────────────────────────┤
│ 服务器动作：                        │
│ 1. 向丘比特推送选择目标UI          │
│ 2. 丘比特选择两名玩家               │
│ 3. 记录情侣关系                     │
│ 4. 判定情侣阵营                     │
│ 5. 向丘比特推送情侣身份             │
│ 6. 向情侣推送："你们是情侣"        │
│    （不告知身份/阵营）              │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ 阶段2：守卫守护                     │
├─────────────────────────────────────┤
│ 服务器动作：                        │
│ 1. 向守卫推送选择守护目标UI         │
│ 2. 守卫选择目标                     │
│ 3. 检查是否连守上一夜的人           │
│   - 前端禁用上局守护目标            │
│ 4. 记录守护目标                     │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ 阶段3：狼人刀杀                     │
├─────────────────────────────────────┤
│ 服务器动作：                        │
│ 1. 向所有狼人推送行动开始           │
│ 2. 开启狼人聊天频道                 │
│ 3. 狼人打字交流                     │
│ 4. 狼人选择刀杀目标                 │
│ 5. 记录刀杀目标                     │
│ 6. 关闭聊天频道                     │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ 阶段4：女巫用药                     │
├─────────────────────────────────────┤
│ 服务器动作：                        │
│ 1. 向女巫推送：                     │
│    - 被刀者座位号                   │
│    - 解药/毒药剩余状态              │
│    - 选择UI                         │
│ 2. 女巫选择是否用药                 │
│ 3. 检查限制：                       │
│    - 不能自救                       │
│    - 每晚只能用一瓶药               │
│ 4. 记录用药情况                     │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ 阶段5：预言家查验                   │
├─────────────────────────────────────┤
│ 服务器动作：                        │
│ 1. 向预言家推送选择查验目标UI       │
│ 2. 预言家选择目标                   │
│ 3. 查询目标身份                     │
│ 4. 向预言家推送查验结果             │
│ 5. 记录查验结果                     │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ 阶段6：夜晚结算                     │
├─────────────────────────────────────┤
│ 服务器动作：                        │
│ 1. 汇总所有夜间行动                 │
│ 2. 执行死亡判定：                   │
│    - 狼刀 + 守卫 + 解药 → 同守同救（奶穿） │
│    - 毒药 → 毒杀                    │
│    - 情侣死亡 → 殉情                │
│ 3. 生成死亡名单                     │
│ 4. 判断遗言权限：                   │
│    - 首夜死亡 → 有遗言              │
│    - 第二夜及以后夜间死亡 → 无遗言  │
│ 5. 推送死亡结果                     │
│ 6. 判断警徽流转：                   │
│    - IF 警长夜间死亡 → 推送传警徽UI │
│    - 警长选择传/不传 → 记录结果     │
└──────────────────┬──────────────────┘
                   │
                   ▼
            白天阶段开始
```

## 三、白天阶段流程（第一天）

```
白天阶段开始
    │
    ▼
┌─────────────────────────────────────┐
│ 步骤1：警长竞选（仅第一天，最先）   │
├─────────────────────────────────────┤
│ 子阶段：                            │
│ 1. 上警 → 玩家点击上警按钮          │
│ 2. 竞选发言 → 按顺序发言            │
│ 3. 退水 → 玩家点击退水按钮          │
│ 4. 投票 → 警下玩家投票              │
│ 5. 平票PK → 平票者再次发言投票      │
│ 6. 警长确定 或 警徽流失             │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ 步骤2：天亮公布                     │
├─────────────────────────────────────┤
│ 服务器动作：                        │
│ 1. 向所有玩家推送"天亮了"           │
│ 2. 推送死亡名单或"平安夜"           │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ 步骤3：遗言处理（首夜死亡玩家）     │
├─────────────────────────────────────┤
│ 服务器动作：                        │
│ 1. 对有遗言权限的玩家推送遗言UI     │
│ 2. 接收遗言                         │
│ 3. 广播遗言给所有玩家               │
│ 4. 死亡玩家报出身份                 │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ 步骤4：白天讨论                     │
├─────────────────────────────────────┤
│ 服务器动作：                        │
│ 1. 警长指定发言顺序                 │
│ 2. 按顺序推送发言UI                 │
│ 3. 接收发言，广播给所有玩家         │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ 步骤5：放逐投票                     │
├─────────────────────────────────────┤
│ 服务器动作：                        │
│ 1. 向所有存活玩家推送投票UI         │
│ 2. 接收投票                         │
│ 3. 统计票数（警长1.5票）            │
│ 4. 判断是否平票                     │
│ 5. 推送投票结果                     │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ 步骤6：放逐后处理                   │
├─────────────────────────────────────┤
│ 服务器动作：                        │
│ 1. 被放逐者发表遗言                 │
│ 2. 被放逐者报出身份                 │
│ 3. 判断特殊技能：                   │
│    - 猎人 → 翻牌，推送开枪UI        │
│    - 白痴 → 翻牌免疫                │
│ 4. 处理猎人开枪                     │
│ 5. 处理警徽流转                     │
└──────────────────┬──────────────────┘
                   │
                   ▼
            胜负判定 或 进入下一夜
```

## 五、阶段转换条件

| 当前阶段 | 触发条件 | 下一阶段 |
|---------|---------|---------|
| 发牌完成 | 所有身份分配完毕 | 第一夜 |
| 夜晚结算完成 | 死亡名单生成 | 白天阶段开始 |
| 白天阶段完成 | 放逐后处理完成 | 胜负判定 |
| 胜负判定 | 未分出胜负 | 下一夜 |
| 胜负判定 | 已分出胜负 | 游戏结束 |

## 六、状态存储内容

服务器需要记录的核心信息：

**玩家状态：**
```
- 座位号
- 身份
- 阵营（好人/狼人/第三方）
- 存活状态
- 是否警长
- 是否情侣
- 情侣另一方座位号
- 技能状态（女巫解药/毒药剩余，守卫上一夜守护目标）
```

**游戏状态：**
```
- 当前是第几天
- 当前阶段
- 警长座位号
- 情侣关系
- 丘比特阵营
- 每天投票记录
- 死亡记录
```
